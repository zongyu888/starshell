# StarShell 打包发布说明书

> 目标：让 **Windows 10 / Windows 11 / macOS** 用户在**无需预装任何 Java 环境**的前提下，通过安装包把程序安装到**自选文件夹**，并依靠安装包**自带的精简 JRE** 启动 exe / app。

---

## 一、整体原理（先理解，再操作）

跨平台"免 Java 安装包"靠两个 JDK 自带工具协作：

| 工具 | 作用 | 产物 |
|------|------|------|
| **jlink** | 从完整 JDK 的 jmods 里抽出应用真正用到的模块，裁剪出一个**精简 JRE** | `target/runtime/`（约 40–60MB，而非完整 JDK 的 300MB+）|
| **jpackage** | 把 [你的 fat-jar] + [精简 JRE] + [图标] 打成**平台安装包** | `.msi`/`.exe`（Win）、`.dmg`/`.pkg`（Mac）、`.deb`/`.rpm`（Linux）|

因为精简 JRE 被**打进安装包**，用户机器上不需要任何 Java——这是"无 jar 环境运行"的核心。你的 `pom.xml` 已经把这条链配好了（jlink → shade → antrun → jpackage）。

### ⚠️ 跨平台打包的第一铁律：jpackage 不能交叉编译

> **Windows 包必须在 Windows 上打；Mac 包必须在 Mac 上打；Linux 包必须在 Linux 上打。**

jpackage 底层调用各平台的原生打包工具（Windows 的 WiX、Mac 的 pkgbuild/dmg、Linux 的 dpkg/rpm），这些工具只存在于对应平台。所以"一份代码出三端安装包"的正确做法是：

- **本机分别构建**：在 Win 机器上打 Win 包，在 Mac 机器上打 Mac 包；或
- **CI 自动化**：用 GitHub Actions 的 `windows-latest` + `macos-latest` + `ubuntu-latest` 三个 runner 各打各的（见第八节）

**不要**指望在 Windows 上一条命令打出 Mac 的 .dmg——做不到。

## 二、当前项目状态盘点

| 项 | 现状 | 是否需要改 |
|----|------|-----------|
| JDK | `F:\java26`（含 jpackage/jlink）✓ | Windows 端不用改 |
| Maven | `F:\apache-maven-3.9.16` ✓ | 不用改 |
| 启动类 | `com.aifinalshell.Launcher`（非 Application 子类，跨平台兼容）✓ | 不用改 |
| JavaFX 依赖 classifier | **写死 `win`** ❌ | Mac 构建必须改成按 OS 自动切换（见第三节）|
| Windows 图标 | `src/main/resources/images/icon.ico` ✓ | 不用改 |
| Mac 图标 | **缺失** ❌ | Mac 构建前需准备 `.icns`（见第四节）|
| jpackage 类型 | `app-image`（绿色文件夹，非安装包）| 改成 `msi`/`dmg` 才是安装包 |
| 精简 JRE（jlink）| 已配 ✓ | 不用改 |

## 三、改造 pom.xml：让一套配置支持三端

当前 `pom.xml` 的 JavaFX 依赖（L31–55）写死了 `<classifier>win</classifier>`，在 Mac 上会缺 native 库（`.dylib`）导致启动崩溃。改造目标：**classifier 跟随当前操作系统自动切换**。

### 第 1 步：在 `<properties>` 里加默认值

```xml
<properties>
    ...既有属性...
    <!-- JavaFX native 平台包，跟随 OS 自动切换（见下方 profiles） -->
    <javafx.classifier>win</javafx.classifier>
    <!-- jpackage 产物类型与图标，跟随 OS 切换 -->
    <jpackage.type>msi</jpackage.type>
    <jpackage.icon>${project.basedir}/src/main/resources/images/icon.ico</jpackage.icon>
</properties>
```

### 第 2 步：把 4 个 JavaFX 依赖的 classifier 改成变量

把 `pom.xml` L31–55 里所有的 `<classifier>win</classifier>` 替换为 `<classifier>${javafx.classifier}</classifier>`（共 4 处：controls / fxml / graphics / swing）。

### 第 3 步：在 `<profiles>` 里加三个按 OS 自动激活的 profile

把现有的 `<profiles>` 段替换/扩充为：

```xml
<profiles>
    <!-- ===== 按 OS 自动切换 JavaFX classifier / 产物类型 / 图标 ===== -->
    <profile>
        <id>platform-windows</id>
        <activation><os><family>windows</family></os></activation>
        <properties>
            <javafx.classifier>win</javafx.classifier>
            <jpackage.type>msi</jpackage.type>
            <jpackage.icon>${project.basedir}/src/main/resources/images/icon.ico</jpackage.icon>
        </properties>
    </profile>
    <profile>
        <id>platform-mac</id>
        <activation><os><family>mac</family></os></activation>
        <properties>
            <javafx.classifier>mac</javafx.classifier>
            <jpackage.type>pkg</jpackage.type>
            <jpackage.icon>${project.basedir}/src/main/resources/images/icon.icns</jpackage.icon>
        </properties>
    </profile>
    <profile>
        <id>platform-linux</id>
        <activation><os><family>unix</family></os></activation>
        <properties>
            <javafx.classifier>linux</javafx.classifier>
            <jpackage.type>deb</jpackage.type>
            <jpackage.icon>${project.basedir}/src/main/resources/images/icon.png</jpackage.icon>
        </properties>
    </profile>

    <!-- ===== dev：只出 fat-jar，不跑 jlink/jpackage（快速开发）===== -->
    <profile>
        <id>dev</id>
        <build>
            <plugins>
                <plugin>
                    <groupId>org.codehaus.mojo</groupId>
                    <artifactId>exec-maven-plugin</artifactId>
                    <executions>
                        <execution><id>jlink</id><phase>none</phase></execution>
                        <execution><id>jpackage</id><phase>none</phase></execution>
                    </executions>
                </plugin>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-antrun-plugin</artifactId>
                    <executions>
                        <execution><id>stage-jpackage-input</id><phase>none</phase></execution>
                    </executions>
                </plugin>
            </plugins>
        </build>
    </profile>
</profiles>
```

### 第 4 步：把 jpackage 的 `--type` 和 `--icon` 改成变量

在 `pom.xml` 的 jpackage execution（约 L264、L287）里：

```xml
<argument>--type</argument>
<argument>${jpackage.type}</argument>
...
<argument>--icon</argument>
<argument>${jpackage.icon}</argument>
```

> 改造后：在 Windows 上跑会自动用 `win` classifier + `msi` + `icon.ico`；在 Mac 上跑会自动用 `mac` classifier + `pkg` + `icon.icns`。**同一份 pom，不同平台各打各的包。**

## 四、Windows 10 / 11 安装包打包

### 4.1 一次性准备

1. **JDK 17+**（你已有 `F:\java26`）
2. **WiX Toolset 3.x**（生成 `.msi` 安装包的必需依赖，jpackage 底层调它）
   - 下载：https://wixtoolset.org/releases/ （选 3.x 稳定版，**不要用 4.x**——jpackage 对 3.x 的 candle.exe/light.exe 兼容最好）
   - 安装后把 WiX 的 `bin` 目录（含 `candle.exe`、`light.exe`）加入系统 `PATH`
   - 验证：新开 cmd 跑 `candle -?` 有输出即可
3. **Maven**（你已有）

### 4.2 打包命令

```bash
mvn clean package -DskipTests -Djdk.home=F:\java26
```

三个参数缺一不可：
- `clean` —— jlink 要求 `target\runtime` 不存在，否则报"目录已存在"
- `-DskipTests` —— 跳测试
- `-Djdk.home=F:\java26` —— PATH 上的 java 是 Oracle javapath 残缺副本（无 jpackage/jlink），必须显式指向完整 JDK

### 4.3 产物

```
target\installer\AI FinalShell-1.0.0.msi      ← 这就是安装包（约 60–90MB）
```

双击 `.msi` → 标准 Windows 安装向导：**用户可自选安装目录** → 开始菜单快捷方式 → 控制面板"程序与功能"可卸载。安装后目标机器**无需任何 Java**，靠安装包内置的精简 JRE 启动。

> 想要 `.exe` 安装包（而非 `.msi`）？把 profile 里的 `<jpackage.type>msi</jpackage.type>` 改成 `exe`。两者都需要 WiX，`.exe` 体验更"原生"，`.msi` 更适合企业部署。

## 五、macOS 安装包打包

### 5.1 一次性准备（在 Mac 上操作）

1. **Mac 机器 + JDK 17+**（用 Homebrew：`brew install openjdk@17`，或装 Oracle JDK）
   - 验证：`/usr/libexec/java_home` 能输出 JDK 路径，`jpackage --version` 有输出
2. **Mac 版图标 `icon.icns`**
   - 从现有 `icon.png` 生成：在 Mac 上用 `iconutil` 或在线工具把 png 转成 icns
   - 放到 `src/main/resources/images/icon.icns`
3. **代码签名证书**（分发给别人时**强烈建议**，否则 Gatekeeper 会拦截）
   - 加入 Apple Developer Program（$99/年），拿到"Developer ID Application"证书
   - 在"钥匙串访问"里导出签名身份

### 5.2 打包命令（在 Mac 终端）

```bash
mvn clean package -DskipTests
```

> Mac 上 `JAVA_HOME` 通常已正确指向完整 JDK，不需要 `-Djdk.home`（如果 `jpackage` 不在 PATH，加 `-Djdk.home=$(/usr/libexec/java_home)`）。

### 5.3 产物

```
target/installer/AI FinalShell-1.0.0.pkg     ← 标准 Mac 安装器
```

双击 `.pkg` → 标准 Mac 安装流程 → 安装到 `/Applications`。安装后**无需任何 Java**，靠内置精简 JRE 启动。

> 想要 `.dmg`（拖拽到 Applications 的磁盘映像）？把 profile 里 `<jpackage.type>pkg</jpackage.type>` 改成 `dmg`。

### 5.4 ⚠️ Mac 分发的关键门槛：签名与公证

未签名的 `.pkg`/`.dmg` 发给别人，双击会被 **Gatekeeper** 拦截："无法打开，因为来自身份不明的开发者"。三种应对：

| 方案 | 适用场景 | 操作 |
|------|---------|------|
| **正式签名 + 公证**（推荐） | 公开分发 | jpackage 加 `--mac-sign --mac-signing-identity "Developer ID Application: 你的名字"`；打包后用 `xcrun notarytool submit` 公证 |
| **Ad-hoc 签名** | 小范围测试 | `codesign --force --deep -s - "AI FinalShell.app"`，仍会被 Gatekeeper 提示，但用户右键"打开"可绕过 |
| **用户手动放行** | 自用/内测 | 告诉用户：右键 app → "打开" → 确认；或 `xattr -d com.apple.quarantine /Applications/AI\ FinalShell.app` |

在 pom 的 jpackage execution 里加 Mac 签名参数（仅 Mac profile 生效）：

```xml
<argument>--mac-sign</argument>
<argument>--mac-signing-identity</argument>
<argument>Developer ID Application: 你的名字</argument>
```

## 六、"无 Java 环境运行"如何保障

已由 `pom.xml` 的 jlink execution 实现（L233–254）。它在 `prepare-package` 阶段执行：

```
jlink --module-path=F:\java26\jmods \
      --add-modules java.base,java.desktop,java.sql,... \
      --output target/runtime --compress 2
```

生成的 `target/runtime/` 是一个**只含必要模块的 JRE**，被 jpackage 整体塞进安装包。用户机器无需预装 Java。

**注意：如果将来用了新的 Java API**（如 RMI、JNDI、HTTP Server），运行时会报 `ClassNotFoundException` / `NoClassDefFoundError`。此时往 pom L244 的 `--add-modules` 列表里补对应模块（如 `java.rmi`、`jdk.httpserver`）即可。

---

## 七、"安装到自选文件夹"如何保障

这是 **installer 类型**（`msi`/`pkg`/`dmg`）天然支持的，无需额外配置：

- **Windows `.msi`**：WiX 生成的安装向导自带"选择安装目录"界面，用户可改路径
- **Mac `.pkg`**：标准 pkg 安装器，默认装 `/Applications`（Mac 习惯不分目录，但用户可手动挪）
- **Mac `.dmg`**：拖拽到 Applications，用户想放哪放哪

> 注意：当前 pom 默认的 `--type app-image` 是**绿色文件夹**（无安装向导、无目录选择），不是安装包。必须改成 `msi`/`pkg`/`dmg` 才有"安装到自选文件夹"的体验。这也是第三节改造的重点。

## 八、CI 自动化（一份代码出三端安装包）

在仓库根加 `.github/workflows/release.yml`，用矩阵在三个平台 runner 上并行打包：

```yaml
name: Release
on:
  push:
    tags: ['v*']
jobs:
  build:
    strategy:
      matrix:
        include:
          - os: windows-latest
            artifact: AI FinalShell-1.0.0.msi
          - os: macos-latest
            artifact: AI FinalShell-1.0.0.pkg
          - os: ubuntu-latest
            artifact: ai-finalshell_1.0.0_amd64.deb
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { distribution: 'temurin', java-version: '17' }
      - name: Install WiX (Windows)
        if: runner.os == 'Windows'
        run: choco install wixtoolset -y
      - name: Build
        working-directory: java-client
        run: mvn clean package -DskipTests
      - uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.os }}-installer
          path: java-client/target/installer/*
```

打 tag 即自动产出三端安装包，下载分发。Mac 签名/公证需在 workflow 里配置 secrets（证书 + Apple ID）。

## 九、常见坑速查

| 现象 | 原因 | 解法 |
|------|------|------|
| `错误: 目录已存在 target\runtime` | jlink 输出目录必须不存在 | 打包前加 `clean`（或在 pom jlink 前加 `<delete dir="target/runtime"/>`）|
| `Failed to delete ...\runtime\lib\modules` | 之前打包的 exe/app 还在运行，锁住 JRE 文件 | 先 `taskkill /F /IM "AI FinalShell.exe"` 关掉运行中的实例再打包 |
| `jpackage not found` / jlink 报错 | PATH 上的 java 是 javapath 残缺副本 | 必须加 `-Djdk.home=F:\java26` 指向完整 JDK |
| Mac 双击 app 闪退 / `NoClassDefFoundError` | JavaFX classifier 还是 `win`，缺 Mac native 库 | 确认第三节 profile 改造生效，Mac 上 classifier 应为 `mac` |
| Mac "无法打开，来自身份不明的开发者" | 未签名，被 Gatekeeper 拦截 | 见第 5.4 节签名/公证；临时让用户右键"打开" |
| 运行时报 `ClassNotFoundException` | jlink 裁剪掉了用到的模块 | 往 pom `--add-modules` 补模块（见第六节）|
| WiX 相关 `candle/light not found` | WiX 未装或不在 PATH | 装 WiX 3.x，把 bin 加 PATH（见 4.1）|
| 安装包太大（200MB+） | 没用 jlink 精简，或把 target/ 全塞进去了 | 确认 jlink execution 生效 + antrun 只暂存 fat-jar（pom L188–209 已配）|
