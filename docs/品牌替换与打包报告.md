## 一、替换范围决策

经与用户确认（AskUserQuestion），替换范围采用 **"仅展示文本"** 策略，即只替换用户可见的展示文本，**保留**以下技术标识不动，以确保不破坏代码结构与运行功能：

| 保留项 | 位置 | 保留原因 |
|--------|------|----------|
| 包名 `com.aifinalshell` | 所有 .java 文件 package/import | 改包名需重构全项目，风险高 |
| 类名 `AiFinalShellApp` | 主类 | 同上 |
| H2 数据库路径 `jdbc:h2:./data/aifinalshell` | `DatabaseManager.java:33` | 改动会导致老用户数据丢失 |
| 密钥常量 `AiFinalShell_2024_Security_Key_v1` | `SecurityUtils.java:137` | 改动会导致已加密数据无法解密 |
| 指代原版 FinalShell 的注释 | `pom.xml:212`, `AiFinalShellApp.java:147` 等 | 说明"仿FinalShell"的出处，属正确引用 |

## 三、保留未改项（重要约束核实）

经 grep 复核，以下 finalshell 字符串 **故意保留**，属预期行为：

1. `SecurityUtils.java:137` — `LEGACY_MACHINE_SECRET = "AiFinalShell_2024_Security_Key_v1"` （AES/GCM 密钥常量，改动将导致历史加密数据无法解密）
2. `DatabaseManager.java:33` — `jdbc:h2:./data/aifinalshell` （H2 数据库文件路径，改动将导致老用户连接信息/历史记录丢失）
3. 所有 `package com.aifinalshell` / `import com.aifinalshell.*` （Java 包名，保留以避免全项目重构）
4. `AiFinalShellApp.java:147` 注释 `类似FinalShell安装包机制` （指代原版 FinalShell 的安装包机制，属正确引用）
5. `pom.xml:212` 附近注释中提及 FinalShell （说明项目为仿 FinalShell）

### 4.2 jpackage 生成 app-image（绿色版）

- 工具: jlink（生成精简 JRE）+ jpackage（--type app-image）
- 产物目录: `target/installer/StarShell/`
- 内容:

  - `app/` — ai-finalshell-1.0.0.jar + StarShell.cfg
